The following reference covers all subcommands and options you can pass to the `cassini` program.

## `cassini process`

Generate a map from a single <GlossaryLink slug="lidar" /> file. The `process` subcommand takes the path to the <GlossaryLink slug="lidar" /> file to process as an argument:

<Tabs syncKey="os">
  <TabItem label="Windows" icon="seti:windows">

```sh frame="none"
cassini process path\to\my\tile.laz
```

  </TabItem>
  
  <TabItem label="macOS" icon="apple">

```sh frame="none"
cassini process ./path/to/my/tile.laz
```

  </TabItem>

  <TabItem label="Linux" icon="linux">

```sh frame="none"
cassini process ./path/to/my/tile.laz
```

  </TabItem>
</Tabs>

### Options

#### `--output-dir` or `-o`

<p>

**Type:** `string`<br />
**Default:** `tile`

</p>

The output directory for the processed LiDAR file.

This subcommand also accepts the [common rendering options](#common-rendering-options).

### Options

<p>

## `cassini render`

Run only the map rendering step for a single tile. The `render` subcommand takes the path to the directory containing the output of the <GlossaryLink slug="lidar" /> processing step as an argument:

<Tabs syncKey="os">
  <TabItem label="Windows" icon="seti:windows">

```sh frame="none"
cassini render path\lidar\step\output\directory
```

  </TabItem>
  
  <TabItem label="macOS" icon="apple">

```sh frame="none"
cassini render ./path/to/lidar/step/output/directory
```

  </TabItem>

  <TabItem label="Linux" icon="linux">

```sh frame="none"
cassini render ./path/to/lidar/step/output/directory
```

  </TabItem>
</Tabs>

### Options

#### `--output-dir` or `-o`

<p>

**Type:** `string`<br />
**Default:** `tile`

</p>

The output directory for the processed tile.

#### `--neighbors` or `-n`

<p>

**Type:** `string[]`<br />

</p>

A list of directories containing the output of the <GlossaryLink slug="lidar" /> processing step for neighboring tiles (see [this post](/what-and-why#the-edges-artifacts-problem)).

This subcommand also accepts the [common rendering options](#common-rendering-options).

## `cassini batch`

Process multiple <GlossaryLink slug="lidar" /> files at once. The `batch` subcommand takes the path to the directory containing the <GlossaryLink slug="lidar" /> files to process as an argument. This argument is optional, its default value is `in`:

```sh frame="none"
cassini batch
```

### Options

#### `--output-dir` or `-o`

<p>

**Type:** `string`<br />
**Default:** `out`

</p>

The output directory for the processed LiDAR files.

#### `--threads` or `-t`

<p>

**Type:** `number`<br />
**Default:** `3`

</p>

In batch mode, you can specify the number of <GlossaryLink slug="thread" label="threads" /> used by Cassini to parallelize the work.

```sh frame="none"
cassini batch --threads 6
```

#### `--skip-lidar`

<p>

**Type:** `boolean`<br />

</p>

The `--skip-lidar` option will skip the <GlossaryLink slug="lidar" /> processing stage of the pipeline. It will only work if you allready ran `cassini` once with the same input files, as it will reuse some temp files.

This subcommand also accepts the [common rendering options](#common-rendering-options).

## `cassini config`

Output a default config.json file, that you can edit to modify the map rendering parameters.

```sh frame="none"
cassini config
```

```json
{
  "yellow_threshold": 0.5,
  "green_threshold_1": 0.2,
  "green_threshold_2": 1.0,
  "green_threshold_3": 2.0,
  "cliff_threshold_1": 45.0,
  "cliff_threshold_2": 55.0,
  "dpi_resolution": 600.0
}
```

## `cassini help`

Print a help message:

```
A software that generates highly accurate topographic maps from LiDAR data. See documentation: https://cassini-map.com. GDAL and PDAL must be installed on the system for this program to work.

Usage: cassini [COMMAND]

Commands:
  process         Generate a map from a single LiDAR file
  lidar           Run only the LiDAR processing step for a single tile
  render          Run only the map generation step for a single tile
  batch           Process multiple LiDAR files at once
  config          Output a default config.json file
  help            Print this message or the help of the given subcommand(s)

Options:
  -h, --help     Print help (see more with '--help')
  -V, --version  Print version
```

The help message is olso printed if cassini is called without any arguments.

## Common rendering options

The following options are available for the `process`, `render` and `batch` subcommands.

### `--skip-vector`

<p>

**Type:** `boolean`<br />

</p>

The `--skip-vector` option will skip the vector processing stage of the pipeline. No file will be downloaded from <GlossaryLink slug="osm" /> and no vector features will be drawn on the map (roads, lakes...).

### `--skip-520`

<p>

**Type:** `boolean`<br />

</p>

The `--skip-520` option will prevent the vector renderer to draw the [520 area that shall not be entered](https://omapwiki.orienteering.sport/symbols/520-area-that-shall-not-be-entered/) symbol.

### `--undergrowth`

<p>

**Type:** `none | merge | 406 | 409`<br />
**Default:** `merge`

</p>

The undergrowth rendering mode. The possible values are:

- `none`: No undergrowth is rendered.
- `merge`: Undergrowth is merged with the vegetation rendering (default mode).
- `406`: Undergrowth is rendered as the [406 vegetation: slow running](https://omapwiki.orienteering.sport/symbols/406-vegetation-slow-running/) symbol.
- `409`: Undergrowth is rendered as the [409 vegetation: walk, good visibility](https://omapwiki.orienteering.sport/symbols/409-vegetation-walk-good-visibility/) symbol.

## Common options

### `--help` or `-h`

Same than `cassini help` or `cassini`

### `--version` or `-V`

Print the `cassini` version

```
cassini 0.4.0
```
